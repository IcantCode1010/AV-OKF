# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## Unreleased

- Moved the manual bulk topic review entry point into the completed document Processing workflow. Review-ready documents now continue directly to **Review and export topics**, while the knowledge bundle explorer remains focused on browsing approved knowledge rather than interrupting document processing.
- Fixed direct approved-OKF chat answers so they now include raw document chunks explicitly coverage-linked to the matched concepts. Open RAG search remains fallback-only, linked chunks stay labeled as raw supporting context, and validation rejects unlinked raw evidence on an `okf_only` route.
- Added the Stage 7C bounded agent-tool layer and Stage 7D dynamic chat knowledge scope. Production retrieval now runs through workspace- and bundle-bound read-only wrappers for approved OKF search, bundle-local graph traversal, coverage-linked/raw RAG, source reads, and mandatory answer validation; calls are capped at eight and persisted in the trace, while model-directed tool choice remains evaluation-only. Chats now persist an ordered 1-10 bundle selection, snapshot bundle IDs and scope version on every turn, expose a searchable `Knowledge sources` selector, search all selected bundles under global evidence caps, retain bundle identity on citations, and disclose conflicting approved exact values instead of silently merging them. Bundle deletion now preserves chat history, promotes a remaining selected bundle, or leaves a read-only zero-source conversation rather than deleting the session.
- Fixed OKF citation return navigation across inline markers, expandable evidence cards, and the Sources side panel. Every approved-topic link now carries the active chat-session path, so **Back to conversation** returns to the originating conversation instead of the conversation list.
- Fixed relation-review navigation so approving, rejecting, swapping, or retrying a candidate refreshes the bundle in place without resetting the page to the top. Discovery and blocked-approval redirects now reopen and anchor the Relation discovery panel.
- Added Relation Discovery V3 without changing deterministic V2 pair generation. Every newly discovered pair now enters a durable, concurrency-one LLM verification queue; the verifier receives exactly one pair, is restricted to the active profile vocabulary, treats concept text as untrusted data, and must return an exact quote from the selected relation source. Confirmed results show confidence, direction, rationale, provider/model, and quote for human review; filtered and failed results remain auditable and retryable as appropriate. Only confirmed candidates participate in pending-edge graph preflight, content hashes force re-verification after edits, direction changes use Swap and reverify, and final approval rechecks evidence before writing the rationale plus quote through the existing exporter. The rollout migration deletes only old pending candidates and preserves approved/rejected rows and all OKF files.
- Improved reviewed relation discovery with profile-versioned stopwords, a two-meaningful-term overlap requirement, actual matched term/tag explanations, stable path-based candidate direction, and explicit query ordering. Bundle discovery and assisted-authoring promotion now share one graph preflight that blocks duplicates, unsafe or inactive targets, type mismatches, directional cycles, symmetric reverse duplicates, and competing supersession while warning on independently justified reverse support/reference edges. Relation suggestions remain pending through both manual promotion and manual approval; final approval can swap direction, re-runs preflight, and re-exports only the selected source concept. Added a dry-run `eval:relations` before/after report command and recorded the first live 31-concept Aviation result plus a 12-candidate human review; the sample still shows deterministic boilerplate/proximity noise, so semantic or broader LLM relation classification remains deferred.
- Added a canonical OKF article-body contract to prevent duplicate presentation framing. Topic enrichment now requests and stores a title-free, Source-free Markdown fragment; export defensively strips exact normalized duplicate H1 headings and model-generated trailing Source sections before adding one canonical title and source block; and both Knowledge readers suppress exported framing while hiding a description only when it exactly equals the first prose paragraph after NFKC/case/punctuation/whitespace normalization. Added an idempotent, dry-run-first `repair:okf-articles` maintenance command whose apply mode requires the exact current report hash, revalidates topic records, preserves approved technical prose and filenames, and re-exports only through the sanctioned exporter. No existing approved bundle content was repaired automatically.
- Added a secure standalone approved-topic citation page for chat. Approved OKF citations now open a focused, same-tab article view with rendered Markdown, bundle/source/page metadata, approval provenance, a validated return link to the originating chat, and an optional link to the full Knowledge Explorer. The route reuses authenticated workspace-scoped bundle lookup, live active-lifecycle and agent-readiness checks, canonical realpath-safe bundle reads, and fail-closed internal Markdown links; unsafe files, cross-workspace bundles, unapproved or inactive concepts, and malicious `returnTo` values cannot expose content or redirect users. Raw RAG citations retain their existing PDF/source behavior.
- Fixed the shared application shell on wide viewports by removing its centered `max-w-7xl` cap and allowing the main column to fill all space beside the fixed sidebar. Data-dense pages such as Documents now use the full available width, while chat applies a centered readability cap only to the inner transcript and composer; the conversation grid and sources panel still fill the shell. The Knowledge Explorer retains its edge-to-edge vertical mode on both legacy and bundle-ID routes.
- Added deterministic topic-continuation recognition before enrichment. Discovery now requires compatible forward/backward markers on adjacent pages, normalizes marker labels against topic titles and evidence headings, resolves unambiguous multi-page chains without extra LLM calls, records ambiguity/evidence metadata under `llm-section-v2`, and preserves valid earlier links when a later boundary is ambiguous. A durable worker-startup reconciliation upgrades existing unapproved topics idempotently: already-proposed continuation pages become authoritative without invalidating completed enrichment, while newly discovered pages require re-enrichment; approved, rejected, and bulk-claimed topics remain untouched.
- Fixed repeatable concept-discovery failures on large documents whose many window candidates required more than the previous 8,000-token consolidation allowance. Window analysis remains capped at 4,000 output tokens, while the final structured document-wide consolidation now receives 16,000 tokens so large valid topic sets can complete instead of ending as `topic_discovery_consolidation_failed` with `No output generated`.
- Fixed PDF upload failures at both request boundaries. Oversized files no longer fail at Next.js with an opaque `413 Body exceeded 26mb limit` before application validation: the Server Action transport now has bounded multipart headroom above the supported 25 MB product limit, and the upload control validates selected size before submission with an actionable split/compress message. Docker builds now carry a unique Next.js deployment ID so tabs opened against an older image hard-refresh instead of submitting stale Server Action IDs; the global error screen also offers an explicit full-page reload recovery.
- Added source-preserving knowledge-bundle deletion to the Knowledge library. Any authenticated workspace member can confirm deletion once; the bundle becomes unavailable immediately while a durable, idempotent BullMQ job removes its topics, authoring state, OKF files, profiles, RAG indexes, relations, chats, and vault entry. Uploaded PDFs, document metadata, custom properties, extracted pages, extraction jobs/logs, and document activity remain as visibly Unassigned documents that can be assigned to another active bundle without automatically restarting authoring or indexing. The final bundle may be deleted without silently recreating General Knowledge. Added authenticated change-aware deletion polling, retry controls, worker-startup reconciliation, unassigned-document guards, and a production-stack E2E probe that verifies source preservation, knowledge cleanup, member access, concurrent-request idempotency, and reassignment.
- Moved permanent document deletion from the Metadata panel into the shared document header so workspace admins can reach it from every document view. Replaced unconditional deletion and bulk-run page refresh loops with authenticated status snapshots that reload only when persisted job state changes, and hardened document-processing polling with a shared active/fingerprint contract plus a one-transition reload guard. This fixes legacy or evaluation documents with no extraction job being interpreted differently by the page and status endpoint, which previously caused an endless two-second reload loop that could interrupt the deletion form. Browser verification permanently deleted two same-named route-evaluation documents in sequence; each durable job completed, its temporary row was removed, and polling stopped after one terminal status request.
- Replaced reversible source-document soft deletion with an admin-only permanent deletion pipeline. A confirmed deletion immediately removes the document from active access, quarantines its OKF concepts, and queues retryable cleanup across MinIO, PostgreSQL, RAG, exported bundle files, relations, embeddings, and document-supported chat answers. Successful cleanup leaves one compact count summary in the bundle log and removes its temporary deletion job. Added `test:e2e:document-deletion`, a disposable Docker-stack probe that verifies workspace authorization, concurrent-request idempotency, approved/rejected/review-pending topic removal, real database cascades, MinIO deletion, bundle-reference cleanup, and assistant-answer tombstoning without touching user content.
- Added a unified document Processing experience. Successful uploads now open a live, domain-neutral workflow panel that derives upload, extraction, metadata discovery, concept discovery, enrichment, relation classification, validation, and review/export states from existing records. A compact header strip keeps active, failed, and action-required work visible from every document panel. Change-aware authenticated polling reloads the current deep link only when the underlying workflow fingerprint changes, including automatic approval progress; clean completion removes the strip and leaves a terminal summary with result links.
- Added bundle-scoped automatic approval and export after assisted authoring. Bundle profiles default automation off and activate changes through versioned admin-controlled drafts. Enabled runs snapshot the profile setting, enrich only high-confidence topics from established source pages, persist skipped topics with deterministic blocker reasons, and enqueue one idempotent sequential bulk run. Topic records and OKF frontmatter now retain human, bulk-human, or automated approval provenance; chat citations distinguish human-approved, automation-approved, mixed-review, and legacy OKF evidence without changing answer eligibility or raw-RAG trust rules.
- Added bundle-scoped bulk approval and OKF export for already-enriched topics. Reviewers can inspect and explicitly select topics across documents, run same-document page-overlap and profile-readiness preflight, confirm the pending OKF embedding estimate once, and follow a durable sequential BullMQ run with per-topic results and retries. Each item reuses the existing enriched-content approval and single-topic exporter, resumes at export after an approval/export split failure, and uses an atomic topic claim so concurrent runs cannot publish the same topic twice.

It now tests the authenticated PDF endpoint across owned, foreign-workspace, nonexistent, and unauthenticated requests; proves foreign and nonexistent responses are indistinguishable; exercises retracted, archived, and deleted-source citation races; verifies exact `KnowledgeGap` positive and negative writes; and checks honest misses expose real bundle/document search counts without citation markers. ### Changed

- Added a durable LLM-assisted knowledge-authoring workflow that starts after extraction and coordinates reversible metadata discovery, document-wide concept discovery, enrichment, profile-vocabulary relation classification, and package validation. Runs expose stage history, retry state, and an explicit cost-confirmation pause above 250,000 estimated input tokens or 25 enrichment candidates. Publication remains human-controlled by default; a bundle admin may explicitly enable high-confidence-only automatic approval/export. Relation approval, archive, retract, and deletion remain human-only.

- Replaced production heading-only topic generation with an automatic durable LLM discovery pipeline. After extraction, every page is analyzed in bounded overlapping windows and candidates are consolidated document-wide into section-level drafts with intelligent titles, concise summaries, source pages, confidence, heading evidence, and boundary rationale. Missing provider keys leave discovery in an explicit waiting state; retries preserve approved/rejected topics. Optional enrichment now produces a separate Markdown article and reviewer-controlled proposed source pages instead of overloading the short topic summary.

Hybrid retrieval now reads OKF before RAG sequentially instead of firing both in parallel, and an `okf_only` route with zero approved evidence downgrades to labeled RAG discovery (explicitly framed as unreviewed, both in the deterministic reply and the LLM prompt) instead of dead-ending. Fixed the LLM answer builder's citation-marker instruction along the way: the prompt described the `[n]` requirement in prose without an example, and `gpt-4o-mini` reliably omitted markers on short discovery answers, always tripping the citation-validation fallback; the prompt now states the rule as a hard requirement with a worked example.

Retrieval results now carry fuller non-persisted `evidence` excerpts (1500 chars) for prompting, separate from the short persisted citation excerpts (240 chars). Live-verified end-to-end with a real OpenAI key: a RAG-routed question produced a synthesized, correctly-cited reply traced as `LLM answer · gpt-4o-mini`.

Re-verified end-to-end with a real OpenAI API key against a full local stack (Postgres/Redis/MinIO, extraction worker, `next dev`): uploaded a fixture PDF, let it extract and index for real, and confirmed `rag_only` chat returns a genuine hybrid-search citation grounded in the actual extracted text.

### Fixed

- Fixed the OpenAI structured-output schema used by guided metadata discovery. The initial `rationale` record emitted unsupported JSON Schema `propertyNames`; it is now a bounded `{field, reason}[]` array, allowing live authoring runs to pass the metadata stage while retaining field-level audit rationale.
- Fixed durable authoring retries in BullMQ. Re-enqueue now removes a prior terminal job with the same deterministic run ID before adding the retry, while active jobs remain deduplicated; previously the database returned to `queued` but the retained failed queue job prevented execution.
- Made every OpenAI structured-output property required for metadata discovery, relation classification, and topic enrichment. Values that may be unknown remain nullable, but no property relies on a JSON Schema default that OpenAI rejects in strict response-format mode.
